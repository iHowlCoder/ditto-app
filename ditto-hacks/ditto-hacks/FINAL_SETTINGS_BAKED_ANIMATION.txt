BAKED ANIMATION
----------------------------------------------------------------

SceneKit automatically plays animations embedded in DAE files.

The code now:
  1. Loads the DAE file with animations
  2. Checks for animations in the model
  3. Prints animation info to console
  4. Plays automatically (no code needed)

Console Output:
  ✅ Found X animations in DAE file
  OR
  ⚠️ No animations found in root node, checking children...


CODE STRUCTURE
----------------------------------------------------------------

// Scale to match your settings
dnaNode.scale = SCNVector3(0.93, 0.93, 0.93)

// Move UP to fill frame
dnaNode.position = SCNVector3(0, 3, 0)

// Apply rotation from your settings
let rotX = 7.0 * Float.pi / 180.0
let rotY = 18.0 * Float.pi / 180.0
let rotZ = -18.0 * Float.pi / 180.0
dnaNode.eulerAngles = SCNVector3(rotX, rotY, rotZ)

// Camera positioned from left-high-far
cameraNode.position = SCNVector3(-16.0, 10.0, 40.0)

// Look at model (shifted up)
cameraNode.look(at: SCNVector3(0, 3, 0))

ANIMATION NOT PLAYING?
----------------------------------------------------------------

If the model doesn't animate:

1. Check console for animation messages
2. Verify DAE file has animations:
   - Open in Blender
   - Check animation timeline
   - Re-export with animations enabled

3. SceneKit auto-plays animations IF:
   - They exist in the DAE file
   - Export settings include animations
   - File format is correct (COLLADA .dae)
